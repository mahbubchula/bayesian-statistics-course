# Contributing

Contributions are welcome if they improve clarity, correctness, reproducibility, or teaching value.

## Useful Contributions

- Fixing typos or unclear explanations
- Adding small reproducible examples
- Improving assignment rubrics
- Adding diagnostic plots
- Adding applied case studies

## Style

- Keep explanations beginner-friendly.
- Use clear notation and define every symbol.
- Prefer reproducible examples using small datasets.
- Keep code readable and avoid unnecessary abstraction.
- Explain statistical interpretation, not only software output.

## Pull Request Checklist

- The contribution fits the course scope.
- Code runs from the repository root.
- New datasets are documented.
- New dependencies are added to `requirements.txt`.
- Markdown links work.
